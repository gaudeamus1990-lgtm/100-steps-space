# 100 шагов вперёд

Первый пробный вариант сайта-приложения для нового партнёра SPACE. Он работает без сервера, базы данных, Google API и сборки проекта.

## Что за что отвечает

- `index.html` — страница партнёра с маршрутом, прогрессом, фильтрами, комментариями и отчётами.
- `admin.html` — демонстрационная админ-панель на тестовых данных.
- `style.css` — внешний вид обеих страниц и адаптация под телефон.
- `steps.js` — список шагов, блоков, задач и ссылок.
- `app.js` — логика страницы партнёра: отметки выполнения, комментарии, отчёты, localStorage.
- `admin.js` — логика демо-админки: таблица партнёров, статистика, подробная карточка.
- `README.md` — инструкция по проекту.

## Как открыть сайт-приложение

Откройте файл `index.html` двойным кликом в браузере. Это страница партнёра.

Откройте файл `admin.html` двойным кликом в браузере. Это демонстрационная админ-панель.

## Где менять шаги

Все шаги находятся в `steps.js`. Каждый шаг выглядит так:

```js
{
  id: "step-001",
  order: 1,
  block: "Блок 1. Настройка рабочего кабинета",
  title: "Установить приложение SPACE",
  shortDescription: "Короткое описание",
  whyItMatters: "Зачем это нужно",
  tasks: ["задача 1", "задача 2"],
  links: [{ title: "Материал", url: "https://example.com", type: "document" }],
  required: true
}
```

## Как заменить временные ссылки на реальные

В `steps.js` найдите нужный шаг и замените `https://example.com` на настоящую ссылку.

## Как добавить новую ссылку к шагу

В поле `links` добавьте ещё один объект:

```js
{
  title: "Название материала",
  url: "https://real-link.ru",
  type: "document"
}
```

Типы ссылок: `video`, `form`, `document`, `chat`, `community`, `calendar`, `other`.

## Как добавить видео-ссылку VK

Добавьте ссылку с типом `video`:

```js
{
  title: "Видео во ВКонтакте",
  url: "https://vk.com/video...",
  type: "video"
}
```

После этого на карточке шага появится кнопка “Открыть видео”.

## Как поменять порядок шагов

В `steps.js` поменяйте число в поле `order`. Лучше также переставить сами блоки кода шагов в нужном порядке, чтобы файл было проще читать.

## Как добавить новый шаг

Скопируйте любой существующий шаг в `steps.js`, вставьте ниже, задайте новый `id`, `order`, `block`, `title`, описания, задачи и ссылки.

## Как удалить шаг

Удалите весь объект шага из массива `STEPS` в `steps.js`. Важно не сломать запятые между соседними шагами.

## Как поменять название блока

В `steps.js` измените поле `block` у всех шагов нужного блока. Например, замените `Блок 4. Обучение` на новое название.

## Как сейчас сохраняется прогресс

Прогресс сохраняется только в браузере конкретного партнёра через `localStorage`: имя, выполненные шаги, комментарии и дата выполнения.

Если партнёр откроет сайт в другом браузере или на другом устройстве, прогресс там не появится. Для общего контроля нужен второй этап с Google Таблицей или сервером.

## Админка сейчас

`admin.html` пока демонстрационная. Она не видит реальный прогресс других людей без Google Таблицы или сервера. Сейчас там показано, как будет выглядеть будущий контроль 50+ партнёров.

## Будущая структура Google Таблицы

Лист `Partners`:

- `partnerId`
- `fullName`
- `phone`
- `messenger`
- `city`
- `status`
- `startDate`

Лист `Steps`:

- `stepId`
- `order`
- `block`
- `title`
- `shortDescription`
- `whyItMatters`
- `tasks`
- `links`
- `required`

Лист `Progress`:

- `partnerId`
- `stepId`
- `done`
- `comment`
- `updatedAt`

## Как позже подключить Google Таблицу

1. Создать Google Таблицу с листами `Partners`, `Steps`, `Progress`.
2. Перенести шаги из `steps.js` в лист `Steps`.
3. Сделать способ идентификации партнёра: например, персональная ссылка с `partnerId`.
4. При отметке шага отправлять `partnerId`, `stepId`, `done`, `comment`, `updatedAt` в лист `Progress`.
5. В админке читать листы `Partners` и `Progress`, считать проценты и показывать реальные данные.
6. Решить вопрос доступа: через Google Apps Script, небольшой сервер или другой безопасный слой между сайтом и таблицей.
